# ADR-001: 습관 체크 날짜 계산 시 타임존 처리 방식

> 날짜: 2026-05-15 | 상태: Accepted

## 컨텍스트

BACKLOG-001 수정 과정에서, 현재 스트릭 계산 로직이 서버 UTC 기준으로 날짜를 판단하고 있음을 확인했다.
한국(UTC+9) 사용자가 23:00~23:59에 습관을 체크하면 서버에서는 다음날 00:00~00:59로 인식되어
당일 스트릭이 끊기는 버그가 발생한다.

## 결정

**클라이언트가 보내는 체크 타임스탬프를 그대로 사용하지 않고,
서버에서 유저의 타임존 오프셋을 기반으로 "로컬 날짜"를 계산해 스트릭을 판단한다.**

유저 테이블에 `timezone` 컬럼을 추가하고, 기본값은 `Asia/Seoul`로 설정한다.

## 이유

### 고려한 대안

1. **클라이언트에서 로컬 날짜를 직접 계산해서 전송**
   - 장점: 서버 로직 단순
   - 단점: 클라이언트 시간을 신뢰해야 함 → 조작 가능성, 앱 버전마다 구현 필요

2. **서버에서 타임존 기반 날짜 계산** ← 채택
   - 장점: 단일 소스 오브 트루스. 클라이언트 신뢰 불필요
   - 단점: DB 스키마 변경 필요 (timezone 컬럼 추가)

3. **UTC 기준 유지 + 허용 버퍼 추가 (예: 자정 +1시간 허용)**
   - 단점: 임시방편. 타임존이 다른 해외 사용자 대응 불가

## 결과

### 긍정적
- 타임존이 다른 해외 사용자도 대응 가능
- 스트릭 계산 로직이 서버 한 곳에 집중

### 부정적 / 주의 사항
- `users` 테이블에 `timezone TEXT DEFAULT 'Asia/Seoul'` 컬럼 마이그레이션 필요
- 기존 유저 레코드에는 기본값이 자동 적용되므로 데이터 마이그레이션 영향 없음
- 향후 유저가 타임존을 직접 변경하는 설정 화면이 필요할 수 있음 (현재는 미지원)
