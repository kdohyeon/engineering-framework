description: "[Workflow 7/9] 테스트 — release 브랜치 기준 milestone 이슈 전체 검증"
---

## 역할
모든 feature가 release 브랜치에 merge된 후, 릴리즈 전 최종 통합 테스트를 진행한다.

## 실행

`.workflow/state.json`에서 현재 릴리즈 정보를 읽는다.

## 1단계: 현황 파악

```bash
# 현재 release 브랜치 확인
cat .workflow/state.json

# milestone 이슈 목록 및 상태 확인
gh issue list --milestone "[버전]" --state all \
  --json number,title,state,labels
```

## 2단계: 테스트 체크리스트 출력

milestone에 속한 각 이슈의 완료 조건을 테스트 항목으로 출력한다:

```
─────────────────────────────────
v1.0.2 릴리즈 테스트 체크리스트
브랜치: release/v1.0.2
─────────────────────────────────

Issue #1: 스트릭 타임존 버그 수정
  [ ] 한국 시간 23:00~23:59 체크 시 당일 스트릭 유지
  [ ] 기존 스트릭 기록 변경 없음

Issue #2: 리마인더 알림 설정
  [ ] 습관별 알림 시간 설정 UI 동작
  [ ] 설정한 시간에 로컬 푸시 알림 발송
  [ ] 알림 on/off 토글 동작

─────────────────────────────────
```

## 3단계: 테스트 결과 수집

각 이슈별로 테스트 결과를 확인한다:

```
Issue #1 테스트 결과: [pass/fail]
Issue #2 테스트 결과: [pass/fail]
```

## 4단계: 결과에 따른 분기

**전체 통과:**
```
─────────────────────────────────
모든 테스트가 통과했습니다. ✓

배포를 진행하려면: /wf-deploy
─────────────────────────────────
```

**일부 실패:**
```
─────────────────────────────────
실패한 항목이 있습니다:
  Issue #2: 알림 발송 실패

해당 이슈를 재구현하려면:
  /wf-implement 2
─────────────────────────────────
```

실패한 이슈를 `/wf-implement`로 수정 후 다시 `/wf-test`를 실행한다.
