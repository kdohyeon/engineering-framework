## 역할
분석된 요건을 **하루 이내 완료 가능한** 단위의 태스크로 분리한다.
태스크 하나 = feature 브랜치 하나 = GitHub Issue 하나.

## 실행

이전 `/wf-analysis` 결과를 바탕으로 태스크를 분리한다.
대화 컨텍스트에 분석 결과가 없으면 사용자에게 분석 결과를 요약해달라고 요청한다.

## 태스크 분리 기준

아래 기준을 적용한다:

- **한 태스크는 한 가지 일만**: UI 구현, API 연동, DB 스키마 변경 등은 분리
- **하루 이내**: 2-4시간 예상이 이상적, 최대 하루
- **완료 조건이 명확**: "구현했다"가 아닌 "X를 하면 Y가 된다"
- **의존 순서 고려**: DB 스키마 → API → UI 순서로 배열

## 출력 형식

```markdown
## 태스크 목록

### TASK-1: [제목]
- 예상 시간: X시간
- 완료 조건:
  - [ ] 조건 1
  - [ ] 조건 2
- 의존: 없음 / TASK-N 완료 후

### TASK-2: [제목]
...
```

## 사용자 확인 (user_confirm)

태스크 목록 출력 후:

```
─────────────────────────────────
태스크 분리가 완료됐습니다. (총 N개)

수정이 필요하면 말씀해주세요.
확정하면 GitHub Issues를 생성합니다.

[Y] 태스크 확정, 백로그 등록 → /wf-backlog
[수정] 특정 태스크를 수정하거나 추가합니다
─────────────────────────────────
```

사용자가 수정을 요청하면 반영 후 다시 확인을 받는다.
