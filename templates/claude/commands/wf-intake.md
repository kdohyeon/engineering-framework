## 역할
새로운 요청/요구사항을 받았을 때 첫 번째로 실행하는 커맨드.
**해석하거나 요약하지 말고** 받은 내용 그대로 저장한다.

## 실행

$ARGUMENTS 가 있으면 그것을 원문으로 사용한다.
없으면 사용자에게 아래를 질문한다:
1. "요청 내용을 그대로 입력해주세요 (수정/요약 없이):"
2. "수신 경로를 선택해주세요: (1) 앱스토어 리뷰  (2) 친구/지인 구두 전달  (3) 직접 사용 중 발견  (4) 기타"

## inbox 파일 생성

수집한 정보로 `docs/inbox/` 폴더에 파일을 생성한다.

파일명: `YYYY-MM-DD-[source]-[slug].md`
- source: appstore | verbal | direct | other
- slug: 내용 핵심 키워드 2-3개 (영어 소문자, 하이픈)
- 예: `2026-05-16-appstore-dark-mode-request.md`

파일 내용:
```markdown
# [날짜] [수신 경로]

## 원문 (그대로)

> [받은 내용 그대로]

## 수신 경로

[앱스토어 리뷰 / 친구 구두 전달 / 직접 사용 중 발견 / 기타]

## 처리 상태

- [x] Inbox 수신 ([날짜])
- [ ] 분석 완료 → /wf-analysis
- [ ] GitHub Issue 생성 → #번호
```

## 완료 후

파일이 생성되면 사용자에게 알린다:
- 생성된 파일 경로
- 다음 단계: `/wf-analysis docs/inbox/[생성된파일명]` 을 실행하라고 안내

## 다음 단계 전환

**자동 전환 (auto)**: 파일 생성 완료 → Analysis 단계 준비 완료.
사용자가 준비되면 `/wf-analysis`를 실행한다.
