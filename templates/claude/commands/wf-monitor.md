## 역할
배포 후 앱 상태를 확인하고 새로운 이슈나 요청을 감지한다.

## 실행

배포 완료 직후 또는 주기적으로 실행한다.

## 모니터링 체크리스트

```
─────────────────────────────────
배포 후 모니터링 체크리스트 (v[버전])
─────────────────────────────────

즉시 확인 (배포 후 1시간):
  [ ] Slack #dev-deploy 채널 - 배포 알림 수신 확인
  [ ] Slack #dev-alerts 채널 - 오류 알림 없음 확인
  [ ] 앱 실행 → 핵심 기능 직접 테스트
  [ ] Supabase 대시보드 - DB 오류 없음 확인

이번 릴리즈 주요 변경 검증:
  [이번 릴리즈에서 변경된 기능들을 직접 사용해보고 확인]

─────────────────────────────────
```

## 장애 감지 시

이상 징후가 발견되면:

```
─────────────────────────────────
장애가 감지됐습니까?

[Y] 긴급 대응 시작 → /wf-hotfix
[N] 정상. 모니터링 계속
─────────────────────────────────
```

## 신규 요청/이슈 접수 시

```
새로운 요청이나 이슈가 있습니까?

[Y] 신규 요건 접수 → /wf-intake
[N] 현재 릴리즈 완료
─────────────────────────────────
```

## 안정화 기간

배포 후 24시간 동안은 주요 지표를 모니터링한다:
- Supabase: 데이터베이스 오류율
- 앱스토어 리뷰: 신규 부정 리뷰
- Slack: 오류 알림

안정화가 확인되면 다음 릴리즈 계획을 시작할 수 있다.
