# Share Tech — 기술 공유 문서 작성

## 프로세스

1. **릴리즈 정보 수집**: 다음을 읽는다:
   - `releases/vX.Y.Z/release-note.md`
   - 포함된 backlog 파일들
   - `git diff main~1 main --name-only` 로 변경된 파일 목록

2. **공유할 내용 추출**: 다음 기준으로 공유 가치가 있는 내용을 선별:
   - 새로운 라이브러리/도구 도입
   - 기존 패턴 대비 다른 접근법 사용
   - 성능 개선이 있는 경우
   - 팀이 모를 수 있는 기술적 결정

3. **공유 문서 생성**: `releases/vX.Y.Z/tech-share.md` 에 생성:

```markdown
# [vX.Y.Z] 기술 공유

**작성일:** YYYY-MM-DD

## 이번 릴리즈에서 주목할 점

### 1. (제목)
**배경:** 왜 이 방식을 선택했는가
**내용:** 무엇을 어떻게 했는가
**코드 예시:**
```언어
// 핵심 코드 스니펫
```
**참고:** (링크 또는 추가 자료)

---
```

4. **저장 경로 안내**: "기술 공유 문서가 `releases/vX.Y.Z/tech-share.md` 에 저장됐습니다. Confluence에 업로드하거나 팀 채널에 공유해주세요." 라고 안내한다.

## 주의사항
- 공유할 내용이 없으면 "이번 릴리즈에서 특별히 공유할 기술적 내용이 없습니다." 라고 알린다.
- 코드 예시는 핵심만 담아 간결하게. 전체 코드 복붙 금지.
