# Intake — 요건/버그 접수

## 프로세스

1. **타입 확인**: 요건(feature)인지 버그(bug)인지 확인한다. 불명확하면 물어본다.

2. **명확화 질문** (모르는 항목만 순서대로 물어본다. 한 번에 하나씩):
   - 배경: 왜 이게 필요한가? 어떤 문제를 해결하는가?
   - 기대 동작: 완료 후 어떻게 동작해야 하는가?
   - 영향 범위: 어떤 기능/모듈/화면에 영향이 있는가?
   - 우선순위: high / medium / low

3. **번호 결정**: `backlog/` 디렉토리를 읽어 현재 가장 높은 번호를 찾고 +1. 파일이 없으면 001부터 시작.

4. **파일명 결정**: `NNN-짧은-영문-설명.md` 형식. 영문, 소문자, 하이픈 사용.

5. **backlog 파일 생성**: `backlog/NNN-title.md` 에 아래 형식으로 생성.

```markdown
---
id: NNN
title: 한글 제목
type: feature
status: ready
release:
priority: medium
created: YYYY-MM-DD (오늘 날짜)
---

## 배경
(명확화된 내용)

## 기대 동작
(명확화된 내용)

## 영향 범위
(명확화된 내용)

## 참고 사항
(있으면)
```

6. **완료 알림**: 생성된 파일 경로와 내용 요약을 사용자에게 알린다.

## 주의사항
- 질문은 한 번에 하나씩. 모든 걸 한꺼번에 묻지 않는다.
- 사용자가 이미 충분한 정보를 제공했으면 추가 질문 없이 바로 파일 생성.
- `backlog/` 가 없으면 먼저 `mkdir backlog` 실행.
