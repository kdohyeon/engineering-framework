# Manager — 진행 상황 & 리스크 점검

## 프로세스

1. **데이터 수집**: 다음 파일들을 읽는다:
   - `.workflow/state.md` — 현재 릴리즈 상태
   - `releases/vX.Y.Z/plan.md` — 릴리즈 계획
   - `backlog/` — in-progress 상태인 파일들 전체

2. **리포트 생성**: 아래 형식으로 리포트를 출력한다:

```
## 릴리즈 vX.Y.Z 진행 상황 (YYYY-MM-DD 기준)

### 전체 진행률
- 전체: N개 중 M개 완료 (X%)

### 태스크별 상태
| 번호 | 제목 | 상태 | 브랜치 |
|------|------|------|--------|
| 001 | 로그인 개선 | in-test | feature/001-... |
| 003 | 결제 버그 | in-dev | feature/003-... |

### 리스크
- ⚠️ (리스크 항목이 있으면 나열, 없으면 "없음")

### 블로커
- 🚫 (블로커가 있으면 나열, 없으면 "없음")

### 일정
- 시작: YYYY-MM-DD
- 현재 상태: (평가)
```

3. **리스크 판단 기준**:
   - 시작일로부터 2주 이상 경과 + 50% 미만 완료: 일정 리스크
   - `closed` 처리된 항목이 있음: 범위 축소됨 (알림)
   - in-dev 상태 태스크가 3개 이상: 병목 가능성

4. **파일 저장 제안**: "리포트를 파일로 저장할까요? (`releases/vX.Y.Z/status-YYYYMMDD.md`)" 라고 물어본다.

## 주의사항
- state.md 가 없거나 비어있으면 "현재 진행 중인 릴리즈가 없습니다. /plan-release 로 시작해주세요." 라고 알린다.
