## 긴급 장애 (Hotfix) 흐름

```
프로덕션 장애 발생
    │
    ├── GitHub Issue 생성 (hotfix 레이블)
    │
    ├── hotfix/N-이름 브랜치 생성 (main 기준)
    │
    ├── 수정 완료
    │
    ├── PR: hotfix/** → main (Backlog / release 스킵)
    │
    ├── git tag v1.1.1 (패치 버전)
    │
    └── GitHub Release 생성
```

---

## 브랜치 생성 기준 요약

```
main           ← 생성 기준 없음 (최초)
release/**     ← main 기준
feature/**     ← release/** 기준   ← 이슈 Milestone 할당 후
hotfix/**      ← main 기준         ← 긴급, Milestone 불필요
develop        ← main 기준         ← 테스트 목적, 언제든 재생성
```

---

## 이슈 상태 흐름

```
생성     →     Backlog     →     Planned     →     In Progress     →     Done
           (Milestone 없음)  (Milestone 할당)    (브랜치 + 개발)      (PR merge)
```
